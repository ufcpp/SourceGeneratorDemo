# Record Equality Generator

This project demonstrates a Source Generator that generates custom equality logic for record types based on attributes.

C# records provide automatic equality comparison based on all properties. However, sometimes you need equality based only on specific "key" properties (like database primary keys or unique identifiers) or want to exclude certain properties from comparison. This generator provides three attributes to customize equality behavior:

- **`[ExplicitKey]`**: Explicitly mark properties to be used for equality (only these properties are compared)
- **`[IgnoreKey]`**: Mark properties to be excluded from equality (all other properties are compared)
- **`[SequenceKey]`**: Use `SequenceEqual` for array/collection comparison instead of reference equality

## Usage

### ExplicitKey - Include Only Specific Properties

Mark properties with `[ExplicitKey]` to include **only** those properties in equality comparison:

```csharp
using RecordEqualityGenerator;

namespace RecordEqualityGenerator;

// Only Name is used for equality comparison
partial record Person
{
    [ExplicitKey]
    public required string Name { get; init; }

    public int Id { get; init; }
    public int Age { get; init; }
}

// Both Id and SKU are used for equality comparison
partial record Product
{
    [ExplicitKey]
    public int Id { get; init; }

    [ExplicitKey]
    public required string SKU { get; init; }

    public required string Name { get; init; }
    public decimal Price { get; init; }
}
```

### IgnoreKey - Exclude Specific Properties

Mark properties with `[IgnoreKey]` to exclude them from equality comparison (all other properties are compared):

```csharp
using RecordEqualityGenerator;

namespace RecordEqualityGenerator;

// Id and Name are used for equality; Age is ignored
partial record Person
{
    public int Id { get; init; }
    public required string Name { get; init; }

    [IgnoreKey]
    public int Age { get; init; }
}
```

### SequenceKey - Compare Collections by Content

Mark array or collection properties with `[SequenceKey]` to use `SequenceEqual` for element-by-element comparison:

```csharp
using RecordEqualityGenerator;

namespace RecordEqualityGenerator;

partial record Data
{
    public int Id { get; init; }

    [SequenceKey]
    public int[] Values { get; init; }
}
```

With `[SequenceKey]`, two `Data` instances are equal if they have the same `Id` and their `Values` arrays contain the same elements in the same order.

**Note:** `[SequenceKey]` can be combined with `[ExplicitKey]` or used with `[IgnoreKey]` mode. It specifies **how** to compare, not **whether** to include a property.

### Generic Records

The generator supports generic records:

```csharp
using RecordEqualityGenerator;

namespace Examples;

// Non-generic record
partial record Pair(
    [property: ExplicitKey] int X,
    [property: ExplicitKey] string Y,
    string Other);

// Generic record with one type parameter
partial record Pair<T>(
    [property: ExplicitKey] T X,
    [property: ExplicitKey] string Y,
    string Other);

// Generic record with multiple type parameters
partial record Pair<T1, T2>(
    [property: ExplicitKey] T1 X,
    [property: ExplicitKey] T2 Y,
    string Other);
```

## Generated Code

The generator creates custom `Equals` and `GetHashCode` methods:

```csharp
// <auto-generated/>
#nullable enable
#pragma warning disable

namespace RecordEqualityGenerator;

partial record Person
{
    public virtual bool Equals(Person? other)
    {
        return ReferenceEquals(this, other) || other is not null && EqualityContract == other.EqualityContract
            && global::System.Collections.Generic.EqualityComparer<string>.Default.Equals(Name, other.Name)
        ;
    }

    public override int GetHashCode()
    {
        var hash = new global::System.HashCode();
        hash.Add(EqualityContract);
        hash.Add(Name);
        return hash.ToHashCode();
    }
}
```

For records with multiple equality keys:

```csharp
// <auto-generated/>
#nullable enable
#pragma warning disable

namespace RecordEqualityGenerator;

partial record Product
{
    public virtual bool Equals(Product? other)
    {
        return ReferenceEquals(this, other) || other is not null && EqualityContract == other.EqualityContract
            && global::System.Collections.Generic.EqualityComparer<int>.Default.Equals(Id, other.Id)
            && global::System.Collections.Generic.EqualityComparer<string>.Default.Equals(SKU, other.SKU)
        ;
    }

    public override int GetHashCode()
    {
        var hash = new global::System.HashCode();
        hash.Add(EqualityContract);
        hash.Add(Id);
        hash.Add(SKU);
        return hash.ToHashCode();
    }
}
```

For records with `[SequenceKey]`:

```csharp
// <auto-generated/>
#nullable enable
#pragma warning disable

namespace RecordEqualityGenerator;

partial record Data
{
    public virtual bool Equals(Data? other)
    {
        return ReferenceEquals(this, other) || other is not null && EqualityContract == other.EqualityContract
            && global::System.Collections.Generic.EqualityComparer<int>.Default.Equals(Id, other.Id)
            && global::System.Linq.Enumerable.SequenceEqual(Values, other.Values)
        ;
    }

    public override int GetHashCode()
    {
        var hash = new global::System.HashCode();
        hash.Add(EqualityContract);
        hash.Add(Id);
        foreach (var item in Values) hash.Add(item);
        return hash.ToHashCode();
    }
}
```

## Behavior

### ExplicitKey Example

With the generated equality methods:

```csharp
var person1 = new Person { Id = 1, Name = "Alice", Age = 30 };
var person2 = new Person { Id = 2, Name = "Alice", Age = 25 }; // Different Id and Age
var person3 = new Person { Id = 1, Name = "Bob", Age = 30 };

person1.Equals(person2); // True - same Name (equality key)
person1.Equals(person3); // False - different Name
person1 == person2;      // True
person1.GetHashCode() == person2.GetHashCode(); // True
```

### IgnoreKey Example

```csharp
var person1 = new Person { Id = 1, Name = "Alice", Age = 30 };
var person2 = new Person { Id = 1, Name = "Alice", Age = 25 }; // Different Age (ignored)
var person3 = new Person { Id = 2, Name = "Alice", Age = 30 };

person1.Equals(person2); // True - Age is ignored
person1.Equals(person3); // False - different Id
```

### SequenceKey Example

```csharp
var data1 = new Data { Id = 1, Values = [1, 2, 3] };
var data2 = new Data { Id = 1, Values = [1, 2, 3] }; // Same content
var data3 = new Data { Id = 1, Values = [1, 2, 4] }; // Different content

data1.Equals(data2); // True - same Id and Values content
data1.Equals(data3); // False - different Values content
```

### Generic Records

```csharp
var pair1 = new Pair<int, string>(1, "A", "Other1");
var pair2 = new Pair<int, string>(1, "A", "Other2");
var pair3 = new Pair<int, string>(2, "A", "Other1");

pair1.Equals(pair2); // True - same X and Y (equality keys)
pair1.Equals(pair3); // False - different X
```
